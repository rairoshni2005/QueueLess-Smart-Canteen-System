# 🚀 Roadmap

This roadmap reflects the current project status and the next planned improvements for QueueLess.

## Current State

### Implemented in the current codebase
- Student and vendor authentication.
- Student menu browsing and search.
- Order creation, queue assignment, and status updates.
- Vendor inventory and orders management.
- Feedback submission with student rewards.
- Analytics endpoint for vendor dashboard.
- AI demand prediction service with fallback.
- MongoDB connection support via `MONGODB_URI`.
- FastAPI AI service at `ai/app.py`.
- Real-time Socket.io updates.

## Near-term Improvements

- Replace frontend hardcoded API base URL with environment-based configuration.
- Add proper admin-only dashboards and role-based UI flows.
- Add order cancellation and refund flows.
- Add payment gateway integration.
- Add vendor-specific food item ownership and filtering.
- Improve analytics with hourly trends and inventory forecasting.

## Mid-term Goals

- Build native mobile apps or PWA support.
- Add push notifications and browser alerts.
- Add rewards redemption and loyalty tiers.
- Add multi-campus vendor/food management.
- Add advanced AI personalization for student recommendations.

## Long-term Vision

- Scale to multiple campuses with centralized vendor analytics.
- Add real-time queue prediction and demand forecasting for multiple locations.
- Add student meal plans and subscription support.
- Support multilingual UI and campus-specific branding.
